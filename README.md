# 簡單呈現如何在兩個Service或兩個Repository，同時Commit或Rollback

- 請先至Program.cs選擇你要用的資料庫，我故意都註解起來了。

## 範例說明

### 例外嘗試方式
1. 若[姓名]包含sigg時，會直接被中斷而不進入UnitOfWork
1. 若推薦人超過兩碼時，則會在UnitOfWork中發生異常，被截取到進而引發Rollback

可觀測兩者發生時，是否會將資料寫入DB

## 功課
 1. 若修改資料時？請自行擴充嘗試
 2. 衍生題型，原提問者並無說是否有多個資料庫連線，那若有多個時你又該如何調整本專案？
 3. 請觀測UseInMemoryDatabase與UseSqlServer()的差別

## 其他提醒
 - 正常情況下，在開發時不應該將邏輯寫至Controller下(例如unitOfWork的執行)，此僅為範例，請建立其他層次去處理邏輯。
 - 此為範例，請對中文命名持包容心態。
 - 如果你沒有.NET 7，可以嘗試降為6，應該大概或許可能不會出錯。
